How it works

SignBridge accepts spoken Arabic or English input via the microphone, or typed text directly, and runs it through an NLP gloss mapping pipeline. The system tokenizes the input, resolves it to an Egyptian Sign Language gloss sequence, and drives an animated sign guide on screen.

The gloss mapping engine handles Arabic morphology — root decomposition, diacritics, verb conjugation — to produce grammatically correct signing sequences. The visual guide updates frame-by-frame so users can follow along in real time without rewinding or pausing.

Use cases

Two-way conversations

Hearing users speak or type; SignBridge converts their message to ESL guidance so the Deaf participant can follow without needing an interpreter.

Public services & counters

Government offices, banks, and hospitals can give staff a tool to instantly communicate with Deaf visitors in structured ESL without extra staff training.

Language learning

Students learning ESL type or speak phrases and see the correct signing sequence, building muscle memory alongside vocabulary comprehension.

Live broadcasting

News channels and live event producers can overlay a sign guide on broadcasts, making live content accessible to the Deaf and Hard-of-Hearing audience.

Technical architecture

Input enters via WebSpeech API (live microphone) or a text field. A preprocessing layer normalizes Arabic Unicode, strips diacritics where appropriate, and segments sentences into translatable clauses. The gloss engine applies a trained bilingual lookup table augmented with a lightweight transformer for OOV handling.

The output gloss sequence drives a pose-based avatar renderer — a rigged skeleton that interpolates between pose keyframes for each sign in the sequence, with configurable playback speed and pause-on-hover support. All processing runs client-side; only the optional TTS step calls a cloud API.
